# ADR-001: Переименование engine/database/ в engine/storage/

## Статус

**Принято**

## Дата

2024

## Контекст

В исходной архитектуре проекта каталог `engine/database/` отвечал за хранение сгенерированных миров в памяти. Однако в корне проекта существует каталог `database/` для файлов СУБД, миграций и дампов.

Это создавало следующие проблемы:

1. **Путаница при навигации** — разработчики могли ошибаться в назначении каталогов
2. **Конфликт имён при импорте** — в коде могли возникать конфликты имён модулей
3. **Неясность в документации** — требовалось уточнять, о каком `database` идёт речь

## Решение

Переименовать `engine/database/` в `engine/storage/`.

### Обоснование названия

- `storage` лучше отражает назначение — хранение данных в памяти
- Избегает конфликта с корневым `database/`
- Соответствует общепринятой терминологии (storage layer, storage engine)

## Последствия

### Положительные

- Устранена путаница между каталогами
- Упрощена навигация по проекту
- Более точное название отражает назначение

### Отрицательные

- Требуется обновление документации
- Изменение в include путях для существующего кода
- Необходимость уведомления команды

## Миграция

Для существующего кода:

```cpp
// Было
#include <engine/database/storage.h>

// Стало
#include <engine/storage/storage.h>
```
