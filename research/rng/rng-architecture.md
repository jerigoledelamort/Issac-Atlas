## Метаданные

| Поле | Значение |
|------|----------|
| Статус | ✅ Подтверждено (частично) |

## Общая цепочка работы RNG

На текущий момент подтверждена следующая последовательность:

```
┌─────────────────────────────────────────────────────────────────┐
│                    ПОЛЬЗОВАТЕЛЬСКИЙ ВВОД                       │
│                    (String Seed / Seed Menu)                    │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  FUN_009EB6B0 — Seed Decoder                              │  │
│  │  ───────────────────────────────────────────────────────  │  │
│  │  • Ожидает строку 9 символов: "XXXX XXXX"                 │  │
│  │  • Base32-декодирование (алфавит: 32 символа)             │  │
│  │  • XOR с константой 0x0FEF7FFD                            │  │
│  │  • Проверка checksum (9-й символ)                         │  │
│  │  • Возвращает: uint32 Internal Seed                       │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  FUN_009EB880 — RNG Initialization                        │  │
│  │  ───────────────────────────────────────────────────────  │  │
│  │  • Принимает готовый uint32 seed                          │  │
│  │  • Инициализирует структуру RNG                           │  │
│  │  • Устанавливает initialized flag                         │  │
│  │  • Сохраняет текущее значение seed                        │  │
│  │  • Устанавливает три параметра сдвига (shift values)      │  │
│  │  • НЕ создаёт новый сид                                   │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  FUN_006EF890 — RNG Step                                  │  │
│  │  ───────────────────────────────────────────────────────  │  │
│  │  • Проверяет, что RNG инициализирован                     │  │
│  │  • Проверяет, что seed != 0                               │  │
│  │  • Обновляет seed по алгоритму XORSHIFT:                  │  │
│  │      x ^= x >> a;                                         │  │
│  │      x ^= x << b;                                         │  │
│  │      x ^= x >> c;                                         │  │
│  │  • Возвращает следующее псевдослучайное значение          │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    ИГРОВАЯ ЛОГИКА                               │
│              (генерация этажей, предметов, врагов)              │
└─────────────────────────────────────────────────────────────────┘
```

### FUN_009EB880 — RNG Initialization

| Характеристика | Значение |
|----------------|----------|
| Назначение | Инициализация внутреннего состояния RNG |
| Вход | uint32 seed (готовый) |
| Выход | void (инициализирует структуру) |
| Статус | ✅ Подтверждено |

**Структура RNG (минимальная):**

```cpp
struct RNG {
    bool initialized;      // Флаг инициализации
    uint32 seed;           // Текущее значение seed
    uint32 shift_a;        // Параметр сдвига 1 = 3
    uint32 shift_b;        // Параметр сдвига 2 = 17
    uint32 shift_c;        // Параметр сдвига 3 = 25
    // ... возможно другие поля
};
```

**Параметры XORSHIFT (подтверждены):**

| Параметр | Значение |
|----------|----------|
| shift1 | 3 |
| shift2 | 17 |
| shift3 | 25 |

**Поведение при seed == 0:**
- При нулевом значении вызывает **аварийную остановку** с сообщением `"RNG Seed is zero!"`.
- НЕ вызывает резервный источник.

**Дополнительное поведение:**
- Заполняет внутренний буфер RNG после инициализации.
- Прогревает внутреннее состояние RNG.

**Ключевые наблюдения:**
- Функция **НЕ создаёт новый сид**
- Передается уже готовый `uint32 seed`
- Устанавливает флаг `initialized`
- Сохраняет три параметра сдвига для XORSHIFT
- При seed == 0 — аварийная остановка с сообщением "RNG Seed is zero!"
- Заполняет внутренний буфер RNG

### FUN_006EF890 — RNG Step

| Характеристика | Значение |
|----------------|----------|
| Назначение | Вычисление следующего состояния RNG |
| Вход | void (использует внутреннее состояние) |
| Выход | uint32 (следующее псевдослучайное значение) |
| Статус | ✅ Подтверждено |
| Алгоритм | XORSHIFT (Marsaglia) |

**Алгоритм:**

```cpp
uint32 RNG_Step() {
    // Проверки
    if (!initialized) return error;
    if (seed == 0) return error;
    
    // XORSHIFT алгоритм
    x ^= x >> a;    // Правый сдвиг
    x ^= x << b;    // Левый сдвиг
    x ^= x >> c;    // Правый сдвиг
    
    seed = x;
    return x;
}
```

**Ключевые наблюдения:**
- Это **классический XORSHIFT** генератор (George Marsaglia)
- Функция **НЕ генерирует первоначальный сид**
- Только вычисляет следующее состояние генератора
- Требует предварительной инициализации через `FUN_009EB880`

### FUN_009EB7F0 — Lua Seeds:SetStartSeed()

| Характеристика | Значение |
|----------------|----------|
| Назначение | Обработчик Lua API Seeds:SetStartSeed() |
| Вход | Строковый пользовательский seed |
| Выход | void (через FUN_009EB880) |
| Статус | ✅ Подтверждено |

**Последовательность:**
1. Принимает строковый пользовательский seed
2. Вызывает `FUN_009EB6B0` (String2Seed / декодер)
3. Получает `uint32` из декодера
4. Передаёт его в `FUN_009EB880` (инициализация RNG)

**Назначение:** Установка пользовательского Custom Seed через Lua API.

> ⚠️ **Это НЕ функция создания StartSeed.** Функция только обрабатывает ввод пользователя и передаёт его в уже существующий pipeline инициализации RNG.

## Неизвестные элементы

### 🔴 StartSeed Source — Источник первоначального сида

**Статус:** ❌ НЕ НАЙДЕН

**Проблема:**
На данный момент не найдена функция, которая создаёт первый `uint32 StartSeed` при создании совершенно нового забега (New Run без ввода пользовательского сида).

**Текущее понимание цепочки:**

Известны обе "половины" преобразования:
- `StartSeed (uint32) → FUN_009EB5B0 → Seed String` (конвертация в строку)
- `Seed String → FUN_009EB6B0 → uint32 → FUN_009EB880 → RNG` (конвертация обратно)

Неизвестным остаётся только участок:
```
??? → StartSeed → Game.StartSeed (+0x4) → FUN_009EB5B0 → SeedString
```

**Известные вызовы FUN_009EB880 (6 всего):**

| # | Функция | Статус исследования | Создание StartSeed |
|---|---------|---------------------|--------------------|
| 1 | `FUN_009EB7F0` | ✅ Подтверждено: Lua Seeds:SetStartSeed() | ❌ Нет (Custom Seed) |
| 2 | `FUN_00958CB0` | ✅ Подтверждено: передаёт существующий seed | ❌ Нет |
| 3 | `FUN_00917A50` | ✅ Подтверждено: восстановление из профиля | ❌ Нет |
| 4 | `FUN_00948FC0` | 🟡 Частично исследована | 🔴 Не подтверждено |
| 5 | `FUN_006F72F0` | ✅ Подтверждено: подготовка забега | ❌ Не найден источник |
| 6 | `FUN_009EB950` | 🔴 Не исследована | — |

**Вывод:** Из шести известных вызовов FUN_009EB880 ни в одной из исследованных функций не найдено подтверждений создания первоначального StartSeed.

**Возможные источники:**
- Системное время (time() / GetTickCount())
- Аппаратный RNG (если доступен)
- Фиксированное значение + модификаторы
- Комбинация вышеперечисленного

**Текущая задача исследования:**
Именно поиск функции, создающей первый `StartSeed`, является основной задачей исследования.

## x32dbg Исследование

### Breakpoint на FUN_009EB6B0

**Дата:** 06.07.2026

**Эксперимент:**
- Установлён breakpoint на `FUN_009eb6b0`
- Запущен обычный New Run

**Результат:**
- Breakpoint **не срабатывает** во время создания обычного нового забега

**Вывод:**
- Обычный New Run **не проходит** через путь декодирования строкового сида
- Это дополнительно подтверждает, что существует **отдельная цепочка генерации StartSeed**
- Путь пользовательского ввода (String → uint32 → RNG) используется только при явном вводе сида пользователем

## Диаграмма состояний RNG

```
┌─────────────┐
│   UNINIT    │ ─────────────────────────────────────┐
│  (start)    │                                       │
└──────┬──────┘                                       │
       │                                              │
       │ FUN_009EB880(seed)                           │
       ▼                                              │
┌─────────────┐                                       │
│  INITIALIZED│ ◄─────────────────────────────────────┘
│  (ready)    │   FUN_006EF890() (step)
└──────┬──────┘   (остается в этом состоянии)
       │
       │ seed == 0 или ошибка
       ▼
┌─────────────┐
│   ERROR     │
│  (invalid)  │
└─────────────┘
```
