# Архитектура проекта Isaac Atlas

## Обзор

Проект Isaac Atlas состоит из двух архитектурно независимых компонентов:

```
isaac-atlas/
├── engine/              ← Библиотека (Isaac Engine)
├── atlas/               ← Приложение (Isaac Atlas)
├── shared/              ← Общие структуры (опционально)
├── game/                ← Материалы игры
├── reverse/             ← Reverse Engineering
├── research/            ← Исследования
├── docs/                ← Документация
├── tests/               ← Тесты
├── scripts/             ← Скрипты
├── tools/               ← Инструменты
├── logs/                ← Логи
├── database/            ← База данных
├── exports/             ← Экспорт
├── temp/                ← Временные файлы
└── third_party/         ← Сторонние зависимости
```

## Isaac Engine

### Назначение

Библиотека для воспроизведения генерации мира The Binding of Isaac: Repentance.

### Модули

#### `engine/core/` — Ядро
- Инициализация движка
- Управление жизненным циклом
- Конфигурация
- Логирование

#### `engine/rng/` — Случайные числа
- Воспроизведение алгоритма RNG игры
- Инициализация seed'а
- Генерация последовательностей
- Сохранение/восстановление состояния RNG

#### `engine/generation/` — Генерация мира
- Генерация этажей
- Размещение комнат
- Выбор врагов и боссов
- Выбор предметов и бонусов
- Специальные комнаты

#### `engine/serialization/` — Сериализация
- Сериализация состояния мира
- Десериализация для сравнения
- Форматы сохранения
- Экспорт/импорт данных

#### `engine/storage/` — Хранение
- Хранение сгенерированных миров в памяти
- Индексация по параметрам
- Поиск и фильтрация
- Управление кэшем

> **Примечание:** Ранее назывался `engine/database/`. Переименован для избежания путаницы с корневым `database/`. См. [ADR-001](docs/decisions/adr-001-rename-engine-database-to-storage.md).

#### `engine/structures/` — Структуры данных
- Структуры данных игры
- Представление комнат
- Представление предметов
- Представление сущностей

#### `engine/verification/` — Верификация
- Сравнение с оригинальной игрой
- Проверка совпадения RNG
- Проверка совпадения генерации
- Генерация отчётов о расхождениях

#### `engine/utils/` — Утилиты
- Вспомогательные функции
- Работа с seed'ами
- Утилиты форматирования
- Математические утилиты

### API Engine

```
// Инициализация
Engine engine;

// Установка seed'а
engine.setSeed(seed);

// Генерация мира
World world = engine.generate();

// Проверка совпадения
bool match = engine.verify(originalWorld);

// Сериализация
Data data = world.serialize();
```

## Isaac Atlas

### Назначение

Пользовательский продукт, использующий Isaac Engine.

### Модули

#### `atlas/app/` — Логика приложения
- Общая бизнес-логика
- Поиск и фильтрация
- Анализ данных
- Кэширование

#### `atlas/cli/` — Командная строка
- Генерация мира по seed'у
- Перебор сидов
- Поиск по параметрам
- Экспорт данных

#### `atlas/gui/` — Графический интерфейс
- Визуализация мира
- Интерактивный просмотр
- Настройки
- Управление

#### `atlas/api/` — REST API
- HTTP сервер
- Эндпоинты для генерации
- Эндпоинты для поиска
- Эндпоинты для данных

#### `atlas/web/` — Веб-интерфейс
- Веб-приложение
- Визуализация
- Поиск и фильтрация
- Адаптивный дизайн

#### `atlas/assets/` — Ресурсы
- Изображения
- Иконки
- Шрифты
- Стили

#### `atlas/config/` — Конфигурация
- Настройки приложения
- Профили
- Локализация
- Системные параметры

## Shared

### Назначение

Общие компоненты, которые могут использоваться обоими компонентами.

### Модули

#### `shared/types/` — Типы данных
- Общие определения типов
- Структуры для обмена данными

#### `shared/constants/` — Константы
- Константы игры
- Глобальные лимиты

#### `shared/protocols/` — Протоколы
- Форматы сериализации
- Протоколы API

#### `shared/utils/` — Утилиты
- Общие вспомогательные функции

## Зависимости между компонентами

```
┌──────────────────────────────────────────────┐
│                    Atlas                      │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐│
│  │  CLI   │ │  GUI   │ │  API   │ │  Web   ││
│  └───┬────┘ └───┬────┘ └───┬────┘ └───┬────┘│
│      └──────────┴────┬─────┴──────────┘     │
│                      │                       │
│              ┌───────▼───────┐               │
│              │     app/      │               │
│              │ (общая логика)│               │
│              └───────┬───────┘               │
│                      │                       │
│         Использует Engine                   │
└──────────────────────┼───────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────┐
│                  Engine                       │
│  ┌────┐ ┌────┐ ┌──────────┐ ┌────────────┐  │
│  │core│ │rng │ │generation│ │serialization│ │
│  └────┘ └────┘ └──────────┘ └────────────┘  │
│  ┌──────────┐ ┌──────────┐ ┌────────────┐  │
│  │ storage │ │structures│ │verification│  │
│  └──────────┘ └──────────┘ └────────────┘  │
│  ┌────────────┐                             │
│  │   utils    │                             │
│  └────────────┘                             │
└──────────────────────────────────────────────┘
                       ▲
                       │
              ┌────────┴────────┐
              │     shared/     │
              │ (общие типы и   │
              │   константы)    │
              └─────────────────┘
```

### Правила зависимостей

1. **Engine** не зависит от Atlas
2. **Atlas** зависит от Engine
3. **Shared** может использоваться обоими
4. **Research/Reverse** не зависят ни от чего

## Принципы архитектуры

1. **Разделение ответственности** — Engine и Atlas независимы
2. **Модульность** — каждый модуль отвечает за свою зону
3. **Тестируемость** — каждый модуль может быть протестирован отдельно
4. **Расширяемость** — новые модули можно добавлять без изменения существующих
5. **Прозрачность** — архитектура следует принципам исследования
