# Архитектура проекта Isaac Atlas

## Обзор

Проект состоит из двух архитектурно независимых компонентов:

```
isaac-atlas/
├── engine/              ← Библиотека (Isaac Engine)
├── atlas/               ← Приложение (Isaac Atlas)
├── research/            ← Исследования и Reverse Engineering
├── docs/                ← Документация
└── tests/               ← Тесты
```

---

## Isaac Engine

Библиотека для воспроизведения генерации мира The Binding of Isaac: Repentance+.

- **НЕ имеет** графического интерфейса
- **НЕ зависит** от Atlas, GUI или веб-интерфейсов
- Содержит **только** логику воспроизведения

### Текущие модули

| Модуль | Назначение | Статус |
|--------|------------|--------|
| `seed_decoder.py` | Декодирование строковых сидов ↔ uint32 (FUN_009EB6B0) | ✅ Реализован |
| `rng/` | Воспроизведение XORSHIFT RNG (shift: 3/17/25) | 🔴 Исследован, не реализован |
| `generation/` | Генерация этажей, комнат, предметов | 🔴 Ожидает исследования |
| `serialization/` | Сериализация состояния мира | 🔴 Ожидает |
| `verification/` | Проверка совпадения с оригиналом | 🔴 Ожидает |

### API Engine

```python
from engine.seed_decoder import decode_seed, encode_seed, is_valid_seed_string

# Декодирование строкового сида в uint32
seed = decode_seed("BMR9 KSHY")  # → 87067453

# Кодирование uint32 в строку
s = encode_seed(87067453)  # → "BMR9 KSHY"

# Проверка валидности
is_valid_seed_string("BMR9 KSHY")  # → True
```

## Isaac Atlas

Пользовательский продукт, использующий Isaac Engine.

- CLI (командная строка)
- GUI (графический интерфейс) — планируется
- REST API — планируется
- Веб-интерфейс — планируется
- Поиск сидов, анализ маршрутов, поиск экстремальных сидов

### Текущие модули

| Модуль | Назначение | Статус |
|--------|------------|--------|
| `cli.py` | Командная строка: decode-seed, encode-seed, validate-seed | ✅ Реализован |
| `gui/` | Графический интерфейс | 🔴 Планируется |
| `api/` | REST API | 🔴 Планируется |
| `web/` | Веб-интерфейс | 🔴 Планируется |

## Зависимости

```
┌──────────────────────────────────┐
│            Atlas                  │
│  ┌──────┐  ┌──────┐  ┌──────┐   │
│  │ CLI  │  │ GUI  │  │ API  │   │
│  └──┬───┘  └──┬───┘  └──┬───┘   │
│     └─────────┴─────────┘       │
│              │                   │
│      Использует Engine           │
└──────────────┼───────────────────┘
               │
               ▼
┌──────────────────────────────────┐
│            Engine                 │
│  ┌────────────┐  ┌────────────┐  │
│  │seed_decoder│  │    rng     │  │
│  └────────────┘  └────────────┘  │
│  ┌────────────┐  ┌────────────┐  │
│  │ generation │  │verification│  │
│  └────────────┘  └────────────┘  │
└──────────────────────────────────┘
               ▲
               │
┌──────────────┴───────────────────┐
│      research/ + docs/            │
│  (исследования, спецификации,     │
│   RE-данные, журнал)              │
└──────────────────────────────────┘
```

### Правила зависимостей

1. **Engine** не зависит от Atlas
2. **Atlas** зависит от Engine через публичный API
3. **Research/Reverse** не зависят ни от чего — это фундамент

---

## Принципы архитектуры

1. **Разделение ответственности** — Engine и Atlas независимы
2. **Модульность** — каждый модуль отвечает за свою зону
3. **Тестируемость** — каждый модуль тестируется отдельно
4. **Расширяемость** — новые модули добавляются без изменения существующих
5. **Исследование прежде кода** — реализация следует за подтверждённым исследованием
