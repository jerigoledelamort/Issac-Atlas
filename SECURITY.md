## Сообщение об уязвимости

Если вы обнаружили уязвимость безопасности:

1. **НЕ создавайте публичный issue**
2. Отправьте отчёт на email: security@isaac-atlas.dev (будет создан)
3. Или используйте GitHub Private Vulnerability Reporting

### Что включать в отчёт

- Описание уязвимости
- Шаги для воспроизведения
- Потенциальное влияние
- Предлагаемое решение (если есть)

### Процесс ответа

1. Подтверждение получения (в течение 48 часов)
2. Оценка уязвимости (в течение 7 дней)
3. Исправление (срок зависит от сложности)
4. Публикация advisory

---

## Типы уязвимостей

### Критические

- Удалённое выполнение кода
- Раскрытие чувствительных данных
- Обход аутентификации

**Время ответа:** 24-48 часов

### Серьёзные

- XSS (для веб-интерфейса)
- CSRF (для API)
- DoS

**Время ответа:** 7 дней

### Умеренные

- Информация о путях
- Недостатки логирования

**Время ответа:** 30 дней

## Безопасность разработки

### Запрещено коммитить

- Пароли и секреты
- API ключи
- Приватные ключи
- Личные данные

### Рекомендации

- Использовать environment variables для секретов
- Регулярно обновлять зависимости
- Проводить security code review
- Использовать статический анализ

- GitHub Dependabot
- OSS-Fuzz (планируется)
- Сканеры уязвимостей в CI
